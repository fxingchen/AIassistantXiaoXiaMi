# 小虾米 v0.0.9 - 主动诊断与修复能力

## 🎯 核心改进

小虾米现在能够**主动诊断和修复问题**，而不只是给出建议！

## ✨ 新增诊断命令

### 1. `/诊断` - 自动测试 API 连接
自动检测配置问题：
- 测试 API Key 是否有效
- 验证 Base URL 是否正确
- 检查模型是否可用
- 返回详细的诊断报告

**使用场景：**
- 遇到 HTTP 400/401/403 错误时
- 不确定配置是否正确时
- 想快速验证连接状态时

### 2. `/配置` - 显示当前配置
查看完整的配置信息：
- 服务商名称
- API Key（部分隐藏）
- Base URL
- 当前模型
- 可用操作列表

### 3. `/切换模型` - 快速切换模型
一键切换到下一个可用模型：
- 自动循环切换
- 保存选择到本地
- 即时生效

**使用场景：**
- 当前模型不可用时
- 想尝试其他模型时
- 遇到频率限制时

### 4. `/重置` - 重置所有配置
清除所有配置，重新开始：
- 清除 API Key
- 重置 Base URL
- 重置服务商
- 打开设置界面

## 🔧 智能错误处理

现在遇到错误时，小虾米会：

1. **自动分析错误类型**
2. **给出具体原因**
3. **提供可执行的命令**（而不是泛泛的建议）

## 🎬 实际使用流程

### 场景1：遇到 HTTP 400 错误

**用户：**帮我写一个函数
**小虾米：**⚠️ HTTP 400 - 请求参数错误

💡 建议操作：输入 `/诊断` 自动检测配置问题

**用户：**`/诊断`
**小虾米：**🔍 正在诊断配置...

❌ **诊断失败**

**错误类型：** invalid_api_key  
**详情：** API Key 无效或已过期

**建议操作：**
- 输入 `/配置` 查看当前设置
- 输入 `/重置` 重新配置
- 输入 `/切换模型` 尝试其他模型

**用户：**`/重置`
**小虾米：**（打开设置界面让用户重新输入 API Key）

### 场景2：遇到频率限制

**用户：**继续
**小虾米：**⚠️ HTTP 429 - 请求频率限制

💡 建议操作：输入 `/切换模型` 尝试其他模型

**用户：**`/切换模型`
**小虾米：**✅ **已切换模型**

**新模型：** DeepSeek-V3  
**模型ID：** deepseek-ai/DeepSeek-V3

可以开始使用了！

## 🎯 设计理念

### 之前的问题
- 只能分析错误，不能修复
- 给出的是泛泛建议，不可执行
- 用户需要手动检查配置
- 错误信息不够具体

### 现在的解决
- 提供可执行的命令
- 自动诊断具体问题
- 一键切换/重置
- 错误信息精确到具体原因
