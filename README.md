# 小虾米 AI 助手

VS Code 侧边栏 AI 聊天插件，基于 SiliconFlow API，支持多模型切换和代码上下文感知。

## 快速开始

1. 点击左侧活动栏的 🐱 图标打开小虾米
2. 首次使用需填入 [SiliconFlow API Key](https://cloud.siliconflow.cn)（免费注册可获得额度）
3. 开始提问

## 直接安装 VSIX

如果当前环境无法本地打包，可以直接使用仓库里已经提交的 VSIX 安装包。

1. 在仓库根目录下载 `claude-chat-0.0.18.vsix`
2. 在 VS Code 中打开“扩展”面板
3. 点击右上角 `...` → `从 VSIX 安装...`
4. 选择下载好的 `claude-chat-0.0.18.vsix`

如果是在命令行安装，也可以执行：`code --install-extension claude-chat-0.0.18.vsix --force`

## 主要功能

### 代码上下文（自动）
- 在编辑器中**选中代码** → 聊天输入框自动出现选区芯片（如 `extension.js:41-51`）
- 只是**打开文件**（无选区）→ 自动出现文件名芯片（如 `extension.js`）
- 发送消息时，芯片对应的代码会自动附加进去
- 点击芯片上的 `×` 可临时移除

### 手动附加上下文
| 操作 | 方式 |
|------|------|
| 发送选中代码 | `Ctrl+Shift+I` 或点击 📎 按钮 |
| 发送整个文件 | 点击 📄 按钮 |
| 粘贴截图 | `Ctrl+V` 直接粘贴 |
| 上传图片 | 点击 🖼 按钮或拖入输入框 |
| 拖入代码文件 | 直接拖文件到输入框 |

### 斜杠快捷指令
输入 `/` 弹出快捷菜单：

| 指令 | 说明 |
|------|------|
| `/解释` | 解释代码作用和逻辑 |
| `/修复` | 找出并修复 Bug |
| `/优化` | 优化代码性能和质量 |
| `/测试` | 生成单元测试 |
| `/审查` | 代码审查 |
| `/文档` | 生成注释和文档 |
| `/翻译` | 翻译注释为中文 |

### 其他
- **多模型切换**：顶部下拉框选择 Qwen2.5-VL-72B、DeepSeek-V3/R1 等模型
- **历史记录**：点击 📋 查看历史对话，最多保存 50 条
- **新对话**：点击 ↺ 开始新对话
- **停止生成**：生成中点击 ■ 停止

### 实验性编程能力（第一阶段）
- **工作区读取**：可列出文件、全文搜索、按行读取目标文件，而不再只依赖当前编辑器内容
- **本地路径只读分析**：支持读取工作区外的本机绝对路径目录和文本文件，用于分析已安装软件或其他本地代码
- **本地路径直接修改**：在明确指定根目录后，可对工作区外本机文件执行结构化补丁，用于修复、升级和定点改写
- **本地修改自动备份/回滚**：工作区外本地补丁会先备份受影响文件；若某一步失败，会自动回滚已完成的前序修改，避免把安装目录改到半截
- **结构化改码**：支持通过结构化补丁直接修改工作区文件，避免把代码误插到当前光标位置
- **精确补丁动作**：支持 `replace`、`insert_before`、`insert_after`、`replace_lines`、`insert_at_line`、`delete_lines`、`regex_replace`、`append`、`prepend`、`create`、`delete`、`rename`，更适合做真实修复和升级
- **基础验证**：修改后可回读 VS Code Problems，辅助判断本次改动是否引入错误
- **验证命令执行**：可先发现项目支持的测试/lint/build/typecheck 命令，再执行白名单内的验证命令，并把真实输出返回给下一轮修复
- **任务优先执行**：如果工作区已有 VS Code 任务，会优先列出并执行现成任务，再在必要时回退到直接命令验证
- **流式执行与停止**：验证命令执行时可实时看到输出，必要时可在聊天里直接停止当前命令
- **失败摘要压缩**：长日志不会原样塞回模型，而是先压缩成结构化错误摘要，再用于下一轮自动修复
- **自动失败分类**：会把验证失败自动标记为语法错、类型错、测试失败、lint 失败、命令不可用等类别，并给出下一步修复策略
- **自动回退止损**：同类失败连续重复时会暂停自动修复，输出最小现状、候选文件和下一步建议，避免盲修
- **中间进度自动续跑**：如果模型只回复“请稍等/我正在处理”这类进度说明，前端会自动追问一次，尽量把工具链路跑完而不是停在半路

当前仍属于第一阶段：更适合小范围、可验证的代码修改；复杂重构仍建议先让它读取相关文件后再逐步执行。验证命令默认仅允许测试、lint、build、typecheck、校验脚本等安全范围内的命令。

## 常见问题

**Q: 提示"API Key 无效"怎么办？**
点击 ⚙ 按钮重新输入 API Key，确认从 [cloud.siliconflow.cn](https://cloud.siliconflow.cn) 复制的是完整的 `sk-` 开头的密钥。

**Q: 选了代码但芯片没出现？**
确保代码文件已在编辑器中打开（不是预览模式），选区后稍等片刻让插件检测到。

**Q: 截图粘贴没反应？**
需要使用支持视觉能力的模型（如 Qwen2.5-VL-72B），其他纯文本模型不支持图片输入。

**Q: 历史记录在哪里存储？**
存储在浏览器 `localStorage` 中（VS Code webview 独立存储），清除插件数据会丢失历史。

**Q: API Key 安全吗？**
Key 仅存储在本地 localStorage，不会上传到任何第三方服务器。
